My name is Zhewei Hu. I am {{ myage }} years old. I was born and grew up in Cixi, Zhejiang Province, China.

In 2016 Spring semester, I received the degree of Master of Science in Computer Science from North Carolina State University. My master thesis is "Scaffolding the Grading Process for Peer Review Assignments: A Pluggable Reputation Web-Service Approach". I am working as a Software Engineer Intern at Offers.com at Austin, Texas during this summer. And I am a Ph.D. student in the Computer Science department of North Carolina State University now.

I started to "play" with Ruby and Rails in Nov. 2014, so it is {{ railsusage }} now. Since that time, I began to work on an open source project named Expertiza. Expertiza is a web application supported by NSF written in Ruby on Rails framework through which students can submit and peer-review learning objects (articles, code, websites, etc). Now I am the core maintainer and contributor of Expertiza and contributed more than 600 commits to Expertiza master branch on GitHub. And I designed and deployed several key features, such as multi-part rubrics, team lazy initialization, automatic review mapping strategy, student suggested topics, staggered-deadline assignment, etc.

I am familiar with the RSpec testing framework. Recently, I refactored and merged several pull requests to increase the test coverage from 28% to 52%. And I also used Rubocop auto-correction functionality to enforce ruby coding guidelines. It improved the Code Climate GPA from 0.9 to 2.5, and the code style of Expertiza is becoming better and better.

I also have experience with DevOps tools. I, together with my teammates, built a DevOps Continuous Integration (CI) and Continuous Deployment (CD) pipeline. We created an Incremental Testing Toolkit to achieve test prioritization and shorten test time by 67%. And we built a Git hook to trigger Behavior Driven Test and Static Analysis Tool to accept or reject a commit. Then we set up Jenkins build server on AWS EC2 to run continuous testing on each Git commit. After that, we deployed code to Canary and Production Server continuously using Ansible. At last, we accomplished load balancing, auto-scaling, and feature flag using Node.js and Redis.

During 2016 summer holiday, I worked as a software engineer intern at Offers.com at Austin, Texas. During this period, I utilized Ruby on Rails, Sequel, and PostgreSQL to build a Key Performance Index (KPI) report for the marketing team. And I also fixed more than 15 issues from JIRA scrum board with Agile methodology. What is more, I implemented Rubocop on 6 projects to enforce coding guidelines (corrected more than 7000 LOC) and created a pre-commit git hook to ensure the quality of commits.

In 2017 summer holiday, I am joining IBM US as an Aspera Software Engineer Intern. My job is related to DevOps. For my first project, I created infrastructure test suites written by InSpec testing framework to verify server environments either converged by Test Kitchen or configured by Chef. Then I set up Cron jobs to monitor infrastructure settings in real-time and send notification emails via SendGrid if exceptions occur. I am working on my second project now, which is related to migrate enterprise server from using Docker Compose to using Kubernetes.
